# A!SK 면접 연습 앱

SK하이닉스 AE 직무 A!SK 비대면 영상 인터뷰 실전 연습 웹앱.

## 앱 목적

A!SK는 7문항(Part1 3개 + Part2 4개), 준비시간 60초, 답변시간 1분/3분의 AI 감독 면접이다. 대본을 읽거나 시선을 돌리면 부정행위로 감지된다. 이 앱은 **실전과 동일한 시간 압박 하에서 자연스러운 구술력**을 기르는 것이 핵심 목표.

---

## 화면 흐름

```
[홈] → [문항 선택] → [준비 카운트다운 60초] → [답변 녹음 타이머] → [결과/재생]
                                                                      ↓
                                                               [전체 모의면접 모드]
```

### 1. 홈 화면

- 앱 타이틀 "A!SK 면접 연습"
- 두 가지 모드 선택:
  - **개별 문항 연습**: 특정 문항만 골라서 반복 연습
  - **전체 모의면접**: 7문항을 실제 순서대로 연속 진행 (중간 이탈 불가)

### 2. 문항 선택 (개별 모드)

- 7개 문항 카드 리스트
- 각 카드에 표시: 문항 번호, 파트(Part1/Part2), 제목, 답변 제한시간
- 카드 탭 → 해당 문항의 준비 단계로 이동

### 3. 준비 단계 (60초 카운트다운)

- 화면 상단: 문항 텍스트 표시
- 중앙: 원형 타이머 (60초 → 0초)
- 하단: 답변 키워드 힌트 (토글로 숨기기/보기 가능, 기본값 = 숨김)
- 60초 종료 시 자동으로 답변 단계 전환 + 녹음 시작
- 수동으로 "답변 시작" 버튼 눌러서 일찍 넘어갈 수도 있음

### 4. 답변 단계 (녹음 + 타이머)

- 화면 상단: 문항 텍스트 (유지)
- 중앙: 답변 타이머 (Part1: 60초, Part2: 180초)
- 녹음 인디케이터 (빨간 점 깜빡임)
- 타이머 종료 시 자동 녹음 중단
- "답변 완료" 버튼으로 일찍 종료 가능
- 30초 미만 답변 시 경고 표시 ("실제 시험에서는 30초 이상 답변 필요")

### 5. 결과 화면

- 답변 시간 표시 (예: "1분 23초 / 3분")
- 녹음 재생 버튼
- 답변 키워드 가이드 표시 (이때는 항상 보임)
- 버튼:
  - "다시 연습" → 같은 문항 준비 단계로
  - "다음 문항" → 다음 번호 문항으로 (전체 모의면접 모드에서)
  - "문항 목록" → 문항 선택 화면으로

### 6. 전체 모의면접 모드

- 7문항 연속 진행 (1번 → 2번 → ... → 7번)
- 각 문항마다 준비(60초) → 답변 자동 진행
- 중간에 결과 화면 없이 바로 다음 문항으로 넘어감 (실전과 동일)
- 모든 문항 완료 후 종합 결과 화면:
  - 각 문항별 답변 시간
  - 각 녹음 재생 가능
  - 총 소요시간

## 문항 데이터

```
문항 1 (Part1, 1분): 자기소개
문항 2 (Part1, 1분): 지원동기
문항 3 (Part1, 1분): 본인의 강점/인성
문항 4 (Part2, 3분): 전공지식 관련 질문
문항 5 (Part2, 3분): 상황판단/문제해결
문항 6 (Part2, 3분): 직무 실무 역량
문항 7 (Part2, 3분): 추가 질문 (산업 맥락/기여 방안 등)
```

각 문항에는 다음 데이터 포함:

- `id`: 문항 번호
- `part`: 1 또는 2
- `title`: 문항 제목
- `question`: 실제 질문 텍스트
- `timeLimitSec`: 답변 제한시간 (초)
- `prepTimeSec`: 준비시간 (60초 고정)
- `keywords`: 답변 키워드 힌트 배열
- `guideNotes`: 답변 가이드 텍스트 (결과 화면에서 표시)

## 답변 키워드 가이드 (문항별)

### 1번 자기소개

- Full Stack AI Memory Creator 시대 + AE 지원 동기
- AI 가속기 회사에서 LLM 추론 메모리 대역폭 병목 체감
- HW 설계(RTL/PPA) ~ 시스템 시뮬레이션(C++ 모델링) 다층적 시각
- 고객 다변화 → 고객 상황 이해 + HW 문제 파악/소통 강점

### 2번 지원동기

- PE(단품 안정화) vs AE(고객 시스템 인증/검증) 차이 인식
- Full Stack Creator 시대에 AE = 고객 접점의 핵심
- cHBM, 고객 다변화로 AE 역할 확대

### 3번 인성/강점

- 컨퍼런스 기획 경험 → 이해관계자 상생
- PPA 최적화 끈기 → 윤리적 기준 타협 없음
- SK CoC (구성원 행복, 이해관계자 상생, 윤리경영) 연결

### 4번 전공지식

- 1T1C DRAM 동작 원리
- DDR4 → DDR5 차이
- Data Eye / Signal Integrity
- HBM/cHBM 기본 개념

### 5번 상황판단

- 상황 파악 → 소통/정보 수집 → 데이터 기반 우선순위 → 협업 해결
- 구체적 사례 연결 가능

### 6번 직무실무

- 원인 추정 → 분석 접근 → 개선 방안 → 유지/피드백
- AE 업무 실제 시나리오 (고객 시스템 이슈 디버깅 등)

### 7번 추가

- "왜 SK하이닉스인가" → cHBM, Creator 전환, AE 역할 변화
- "마지막으로 하고 싶은 말" 대비

## 기술 스택

- React 18 + Vite 5
- Tailwind CSS 3
- Web Audio API (MediaRecorder) — 녹음/재생
- state만으로 관리 (별도 상태관리 라이브러리 불필요)
- localStorage 없음 — 세션 내 메모리로만 운영 (Vercel 배포 호환)

## 주의사항

- **Bringup 벡터 / DRAM SWEEP Vector 관련 내용 절대 포함하지 않기** — 실제 경험 아님
- 키워드 힌트는 연습 보조용이지, 실전에서 보고 읽으면 부정행위
- 녹음 권한은 브라우저에서 최초 1회 요청됨 (HTTPS 필수 → Vercel OK)
