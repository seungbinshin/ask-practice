# A!SK 면접 연습 앱 설계 문서

> SK하이닉스 AE 직무 A!SK 비대면 영상 인터뷰 실전 연습 웹앱

## 목적

A!SK는 7문항(Part1 3개 × 1분 + Part2 4개 × 3분), 준비시간 60초의 AI 감독 비대면 영상 면접이다. 대본을 읽거나 시선을 돌리면 부정행위로 감지된다. 이 앱은 **실전과 동일한 시간 압박 하에서 자연스러운 구술력**을 기르는 것이 핵심 목표.

---

## A!SK 공식 구조 (응시 가이드 기준)

| 파트 | 문항 | 내용 | 답변시간 |
|---|---|---|---|
| Part1-1 | 1분 소개 | 지원자 님만의 이야기를 자유롭게 | 1분 |
| Part1-2 | 나의 경험 | 경험 기반 구체적 답변 | 1분 |
| Part1-3 | 팀 워크 | 제시 상황에서 어떻게 행동할 것인지 | 1분 |
| Part2 | 직무 지식 ×4 | AE 직무 관련 기초 수준 문제, 랜덤 출제 | 각 3분 |

- 모든 문항 준비시간 60초 동일
- 모든 답변 30초 이상 필수
- 영상 면접 응시 시간 약 40분

### 후기 반영 사항

안내문 질문과 실제 질문이 다를 수 있음:
- "자기소개" → "왜 이 회사에 적합한지, 목표는"으로 변형
- "팀워크" → 갈등 상황, 윤리적 딜레마로 변형
- 과거에는 Part1이 4문항이었으나 현재 3문항으로 축소

따라서 Part1도 카테고리별 질문 풀을 두고 variation 대비.

## 화면 흐름

```
[홈] → [모드 선택]
         ├─ [개별 연습] → [카테고리/문항 선택] → [준비 60초] → [답변 녹음] → [결과/재생]
         └─ [전체 모의면접] → [준비 60초] → [답변] → (×7 자동 진행) → [종합 결과]
```

### 화면 목록

| 화면 | 컴포넌트 | 역할 |
|---|---|---|
| 홈 | Home | 앱 타이틀 + 모드 선택 (개별/모의면접) |
| 문항 선택 | QuestionSelect | Part1(3카테고리) + Part2(직무) 카드. Part2는 "랜덤 1문항 뽑기" |
| 준비 | PrepTimer | 60초 원형 카운트다운 + 문항 텍스트 + 키워드 힌트 토글 |
| 답변 | AnswerRecorder | 녹음 + 답변 타이머 (Part1: 60초, Part2: 180초) |
| 결과 | Result | 답변 시간, 녹음 재생, 키워드 가이드 + 답변 골격 |
| 모의면접 | MockExam | 7문항 자동 연속 진행 컨트롤러 |
| 종합 결과 | MockResult | 각 문항별 시간/재생 + 총 소요시간 |

### 출제 로직

- **개별 모드**: 카테고리 선택 → 해당 풀에서 랜덤 1개
- **전체 모의면접**:
  - Part1: 각 카테고리(intro, experience, teamwork) 풀에서 랜덤 1개씩 = 3문항
  - Part2: 직무 풀 22개에서 중복 없이 랜덤 4개 = 4문항
  - 총 7문항, Part1 → Part2 순서로 진행

### 답변 가이드 수준

각 문항에 **키워드 + 답변 골격** 포함:
- `keywords`: 핵심 키워드 배열 (준비 단계 힌트 토글)
- `guideNotes`: 2~3줄 답변 흐름 (결과 화면에서 표시)

예시 (tech-1):
```
keywords: ['1T1C', 'Capacitor 전하 저장', '누설 → Refresh', 'ACTIVATE → READ/WRITE → PRECHARGE']
guideNotes: '1T1C 구조(1 Transistor + 1 Capacitor) 설명 → Capacitor에 전하 저장/누설되므로 "Dynamic" → Refresh 필요(64ms 이내 모든 row) → 4대 동작(ACTIVATE, READ/WRITE, PRECHARGE, REFRESH) 흐름'
```

## 컴포넌트 상세

### 파일 구조

```
src/
├── main.jsx
├── App.jsx
├── index.css
├── data/
│   └── questions.js
└── components/
    ├── Home.jsx
    ├── QuestionSelect.jsx
    ├── PrepTimer.jsx
    ├── AnswerRecorder.jsx
    ├── Result.jsx
    ├── MockExam.jsx
    ├── MockResult.jsx
    └── CircleTimer.jsx
```

### App.jsx — 상태 관리

```js
{
  screen: 'home' | 'select' | 'prep' | 'answer' | 'result' | 'mock' | 'mockResult',
  mode: 'individual' | 'mock',
  currentQuestion: { id, question, keywords, guideNotes, timeLimitSec, prepTimeSec, category },
  recordings: Map<questionId, { blob, duration }>,
  // 모의면접 전용
  mockQuestions: [...],  // 7문항 세트
  mockIndex: 0,          // 현재 진행 중인 문항 인덱스
  mockPhase: 'prep' | 'answer'
}
```

### PrepTimer

- 원형 타이머 (CircleTimer) 중앙, 숫자 크게
- 문항 텍스트 상단 고정
- 키워드 힌트: 토글 버튼 (기본 = 숨김)
- 60초 종료 → 자동 AnswerRecorder 전환 + 녹음 시작
- "답변 시작" 버튼으로 일찍 넘어가기 가능

### AnswerRecorder

- 문항 텍스트 유지
- 빨간 점 깜빡임 (recording-pulse) + "녹음 중"
- 타이머: Part1 → 60초, Part2 → 180초
- 색상 변화: ≤30초 → 노란색, ≤10초 → 빨간색
- 30초 미만 종료 시 경고 메시지
- 시간 종료 → 자동 녹음 중단
- "답변 완료" 버튼으로 일찍 종료

### MockExam

- 시작 시 7문항 세트 생성
- 문항 간: 준비 → 답변 → 바로 다음 문항 준비 (중간 결과 없음, 실전 동일)
- 상단 진행 바: "3 / 7"
- 모든 문항 완료 → MockResult

### Result / MockResult

- 답변 시간 표시 (예: "1분 23초 / 3분")
- 녹음 재생 버튼
- 키워드 + 답변 골격 가이드 표시
- 30초 미만이면 경고 배지
- MockResult: 각 문항별 시간/재생 + 총 소요시간
